# 个人平台 — 优化后框架说明

## 一、功能分区总览

```
┌─────────────────────────────────────────────────────────────┐
│  主界面（入口）                                                │
│  核心：创建 → 二选一「待办」 / 「随记」                          │
└─────────────────────────────────────────────────────────────┘
           │
           ├── 创建 → 待办（完整表单）
           └── 创建 → 随记（随记表单）

┌─────────────────────────────────────────────────────────────┐
│  次界面 1：分类查看                                            │
│  按「分类标签」罗列待办，常见 UI：标签列表 + 该标签下待办列表      │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  次界面 2：时间轴                                              │
│  以日历/时间为轴，串起待办（ddl）与随记（记录时间）               │
│  UI：日历视图 或 时间线列表                                     │
└─────────────────────────────────────────────────────────────┘
```

- **主界面**：以「创建」为主，创建内容分为 **待办** 与 **随记** 两类。
- **分类查看**：按标签筛选/分组，主要面向待办（标签为核心维度）。
- **时间轴**：以时间为主参数，统一展示待办与随记。

---

## 二、数据模型

### 2.1 待办（一条「事件」）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| type | `'todo'` | 固定，用于区分类型 |
| title | string | 事件名称 |
| ddl | string (ISO 8601) | 截止时间 |
| importance | 1 \| 2 \| 3 \| 4 \| 5 | 重要程度（1 最低，5 最高） |
| tags | string[] | 分类标签（用于「分类查看」） |
| description | string | 具体描述 |
| location | string | 地点 |
| relatedPeople | string[] | 相关人员 |
| attachments | Attachment[] | 图片或其他附件 |
| hasSubEvents | boolean | 是否含有子事件 |
| subEvents | TodoItem[] | 子事件列表（嵌套待办） |
| createdAt | string (ISO) | 记录创建时间 |

### 2.2 随记

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| type | `'quicknote'` | 固定 |
| source | string | 来源 |
| content | string | 个人思考内容 |
| time | string (ISO) | 记录时间（用于时间轴） |
| location | string | 地点 |
| attachments | Attachment[] | 图片或其他附件 |
| createdAt | string (ISO) | 创建时间 |

### 2.3 附件（待办与随记共用）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 文件名/标题 |
| type | string | MIME 或扩展名 |
| dataUrl | string | Base64 或 Data URL（本地存储时） |

---

## 三、界面与路由

| 路由 | 名称 | 说明 |
|------|------|------|
| `/` | 主界面 | 创建入口：选择「待办」或「随记」→ 进入对应创建表单；可带简短「最近」预览 |
| `/create/todo` | 创建待办 | 待办完整表单（所有字段） |
| `/create/quicknote` | 创建随记 | 随记表单 |
| `/item/:type/:id` | 详情 | 查看/编辑单条待办或随记 |
| `/tags` | 分类查看 | 按标签罗列待办，支持多标签筛选 |
| `/timeline` | 时间轴 | 日历或时间线，按时间展示待办 + 随记 |

---

## 四、UI 范式约定

- **分类查看**：左侧或顶部为「标签列表」，点击某标签后在右侧/下方展示该标签下的待办列表；支持「全部」「未分类」。
- **时间轴**：采用常见日历或时间线范式——按日/周/月切换，每条待办按 ddl、随记按 time 落在对应日期；点击进入详情或快速编辑。

---

## 五、存储与状态

- 本地：`todos: TodoItem[]`、`quickNotes: QuickNoteItem[]` 持久化（如 localStorage）。
- 时间轴：前端合并 `todos` 与 `quickNotes`，按时间排序展示。
- 分类查看：仅使用 `todos`，按 `tags` 聚合/筛选。

以上为优化后的统一框架，实现时按此数据模型与路由落地即可。
