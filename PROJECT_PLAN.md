# 个人待办/日程/记事/备忘 PWA — 进度安排与输入清单

## 快速启动

```bash
cd "e:\dewdew"
npm install   # 已执行可跳过
npm run dev   # 开发：浏览器打开 http://localhost:5173
```

- **本机访问**：`http://localhost:5173`
- **手机同 WiFi 访问**：`http://你的电脑IP:5173`（如 `http://192.168.1.100:5173`）
- **生产构建**：`npm run build` → 输出在 `dist/`，可部署到任意静态托管
- **iPhone 添加主屏**：用 Safari 打开上述地址 → 分享 → 「添加到主屏幕」

## 一、进度安排（按阶段执行）

| 阶段 | 内容 | 预计耗时 | 产出 |
|------|------|----------|------|
| **阶段 1** | 项目初始化：Vite + React + PWA 插件、响应式布局、路由 | 0.5 天 | 本地可运行、手机可访问的空白 PWA |
| **阶段 2** | 待办事项：列表、新增/完成/删除、本地持久化 | 0.5 天 | 可用的待办模块 |
| **阶段 3** | 日程规划：日历视图、事件 CRUD、日/周切换 | 0.5 天 | 可用的日程模块 |
| **阶段 4** | 记事与备忘：记事列表与编辑、备忘快速录入与置顶 | 0.5 天 | 四模块齐全 |
| **阶段 5** | PWA 完善：manifest 图标/名称、离线缓存、添加到主屏引导 | 0.5 天 | 可「添加到主屏幕」的完整 PWA |
| **阶段 6**（可选） | 部署：选托管平台、配置域名/HTTPS、在 iPhone 上添加主屏 | 0.5 天 | 线上可用的 URL，iPhone 主屏图标 |

## 二、您需要输入/决定的内容

在对应阶段前或开发过程中，请准备好以下信息（有默认值的可直接说「用默认」）：

### 2.1 基础信息（阶段 1 前）

| 序号 | 需要您输入 | 说明 | 默认/示例 |
|------|------------|------|-----------|
| 1 | **应用名称** | 显示在手机主屏图标下方的文字 | 「个人助手」 |
| 2 | **主题色** | 主按钮、强调色、状态栏等 | 如 `#2563eb`（蓝）或「保持你定」 |
| 3 | **是否要登录/多账号** | 仅本机使用可不做；多设备同步则需要 | 先「仅本机，不做登录」 |

### 2.2 功能偏好（阶段 2～4 中按需确认）

| 序号 | 需要您输入 | 说明 | 默认 |
|------|------------|------|------|
| 4 | 待办是否需要「分类/标签」 | 是/否 | 否，先做单列表 |
| 5 | 日程是否需要「重复事件」 | 如每周、每月重复 | 否，先做单次事件 |
| 6 | 记事格式 | 纯文本 / 简单 Markdown | 纯文本 |
| 7 | 备忘是否需要「过期提醒」 | 到时浏览器通知 | 否，先只做置顶 |

### 2.3 部署相关（阶段 6 前）

| 序号 | 需要您输入 | 说明 | 默认 |
|------|------------|------|------|
| 8 | **部署方式** | 仅本机 / 局域网 / 公网 | 先「本机 + 局域网」 |
| 9 | **公网托管**（若选公网） | Vercel / Netlify / 自建服务器等 | 可后续再选 |
| 10 | **域名**（若用公网且要自定义） | 如 `todo.mydomain.com` | 可先用平台免费二级域 |

## 三、当前默认约定（未输入时按此执行）

- **应用名称**：个人助手  
- **仅本机使用**：不实现登录，数据存浏览器本地（localStorage/IndexedDB）  
- **待办**：单列表，支持完成/未完成、删除  
- **日程**：单次事件，日/周视图  
- **记事**：纯文本，列表 + 详情编辑  
- **备忘**：短文本，支持置顶  
- **主题色**：蓝色系 `#2563eb`  

---

## 四、每阶段结束时您需要做的

- **阶段 1 后**：在电脑浏览器打开项目，用 Chrome 开发者工具切换为手机尺寸，确认布局正常；确认 PWA 可安装（地址栏出现「安装」或「添加到主屏幕」提示）。  
- **阶段 2～4 后**：在手机同一 WiFi 下用 `http://你电脑IP:5173` 访问（或部署后用 HTTPS 地址），在 Safari 中「添加到主屏幕」，确认各模块可用。  
- **阶段 6 后**：用最终 HTTPS 地址在 iPhone Safari 添加主屏，确认图标、全屏、离线可用。
